QuickNotes - Troubleshooting Guide

Note: This guide will help you troubleshoot issues with VPN and proxy settings that might be interfering with localhost connections.

VPN Troubleshooting

1. Check if a VPN is Active

VPNs can interfere with localhost connections by routing all traffic through their servers or by changing your network configuration.

  1. Check your menu bar or system tray for VPN icons
  2. Look in System Preferences > Network for any VPN connections
  3. Try temporarily disconnecting from your VPN

2. VPN Split Tunneling

If you need to keep your VPN connected, check if it supports split tunneling:

  • Split tunneling allows you to route some traffic through your regular network connection
  • Configure your VPN to exclude localhost or 127.0.0.1 from the VPN tunnel
  • Check your VPN's settings or preferences for split tunneling options

Proxy Troubleshooting

1. Check System Proxy Settings

Proxy servers can intercept and potentially block localhost connections.

  1. On macOS: Go to System Preferences > Network > Advanced > Proxies
  2. On Windows: Go to Settings > Network & Internet > Proxy
  3. Check if any proxy servers are enabled
  4. Try temporarily disabling all proxy settings

2. Check Browser Proxy Settings

Some browsers have their own proxy settings that override system settings:

  • Chrome: Settings > Advanced > System > Open your computer's proxy settings
  • Firefox: Options > General > Network Settings > Settings
  • Safari: Preferences > Advanced > Proxies: Change Settings

3. Check Environment Variables

Proxy settings can also be set as environment variables:

  • Check for HTTP_PROXY, HTTPS_PROXY, or NO_PROXY variables
  • In Terminal, run: env | grep -i proxy
  • If found, try temporarily unsetting them: unset HTTP_PROXY

Alternative Connection Methods

1. Use IP Address Instead of Hostname

Try using 127.0.0.1 instead of localhost:

  • Example: http://127.0.0.1:9090 instead of http://localhost:9090
  • This bypasses hostname resolution which can be affected by VPNs

2. Try Different Ports

Some networks or security software block specific ports:

  • Common development ports like 3000, 8000, 8080 might be blocked
  • Try using less common ports like 9090, 7000, or 4000

3. Open Files Directly

For static websites, you can open the HTML files directly:

  • Navigate to the QuickNotes folder in your file explorer
  • Double-click on index.html to open it directly in your browser
  • Note: Some features might not work correctly when opened this way

Warning: Be cautious when changing network settings. Make note of your original settings before making changes so you can restore them if needed.
