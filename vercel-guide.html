Deploying QuickNotes to Vercel

Note: This guide will walk you through deploying your QuickNotes application to Vercel, making it accessible online to anyone.

What is Vercel?

Vercel is a cloud platform for static sites and serverless functions that enables developers to host websites and web services that deploy instantly and scale automatically â€“ all without any configuration.

Vercel's deployment workflow

Why Deploy QuickNotes to Vercel?

  • Free hosting - Vercel offers a generous free tier
  • Global CDN - Your app will be fast for users worldwide
  • HTTPS by default - Secure connections for all users
  • Continuous deployment - Updates automatically when you push changes
  • Custom domains - Use your own domain name if desired

Step-by-Step Deployment Guide

1 Sign up for Vercel

Go to vercel.com and sign up for an account. You can sign up using GitHub, GitLab, or Bitbucket.

Vercel sign up page

2 Create a Git Repository

Push your QuickNotes project to a Git repository on GitHub, GitLab, or Bitbucket:

# Navigate to your QuickNotes directory
cd MiniJobb/QuickNotes

# Initialize a new Git repository
git init

# Add all files
git add .

# Commit the files
git commit -m "Initial commit"

# Add your remote repository URL
git remote add origin <your-repository-url>

# Push to the repository
git push -u origin main

3 Import Your Repository

In the Vercel dashboard, click "New Project" and select your repository:

Importing a Git repository in Vercel

4 Configure Your Project

Configure your project settings:

  • Framework Preset: Select "Other"
  • Root Directory: Leave as "." (or specify the path if QuickNotes is in a subdirectory)
  • Build Command: Leave empty
  • Output Directory: Leave empty
Configuring project settings

5 Deploy

Click the "Deploy" button and wait for the deployment to complete:

Successful deployment

6 Access Your Deployed App

Once deployment is complete, you'll get a URL where your app is hosted (e.g., https://quicknotes-abc123.vercel.app).

Click "Visit" to see your live QuickNotes application!

Managing Your Deployment

Updating Your App

To update your app, simply push changes to your Git repository. Vercel will automatically deploy the updates:

# Make changes to your files
git add .
git commit -m "Update app"
git push

Adding a Custom Domain

To use your own domain name:

  1. Go to your project in the Vercel dashboard
  2. Click "Settings" > "Domains"
  3. Add your domain and follow the instructions

Viewing Analytics

Vercel provides analytics for your deployed app:

  1. Go to your project in the Vercel dashboard
  2. Click "Analytics" to see visitor statistics and performance data

Important Note: Since QuickNotes uses localStorage for data storage, each user's notes will be stored in their own browser. Notes created on one device won't be accessible from another device.

Troubleshooting

Deployment Failed

If your deployment fails:

  1. Check the build logs in the Vercel dashboard
  2. Make sure all file paths in your HTML, CSS, and JavaScript files are relative
  3. Verify that your vercel.json configuration is correct

Assets Not Loading

If images or assets aren't loading:

  1. Check that the paths are correct
  2. Make sure the files are included in your Git repository
  3. Verify that the paths in your HTML/CSS files match the actual file structure
