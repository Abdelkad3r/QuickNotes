Git Commands for QuickNotes

This page provides the Git commands you need to run to push your QuickNotes project to GitHub. You can copy each command and paste it into your terminal.

Note: Make sure you're in the QuickNotes directory before running these commands.

Step 1: Navigate to the QuickNotes Directory

First, make sure you're in the QuickNotes directory:

cd MiniJobb/QuickNotes

Step 2: Initialize Git

Initialize a new Git repository in this directory:

git init

Expected output: Initialized empty Git repository in .../MiniJobb/QuickNotes/.git/

Step 3: Add Your Files

Add all the files in the directory to Git:

git add .

This command doesn't produce any output if successful.

Step 4: Configure Git User (if needed)

If this is your first time using Git, you may need to configure your user name and email:

git config user.name "Your Name"
git config user.email "your.email@example.com"

Replace "Your Name" and "your.email@example.com" with your actual name and email.

Step 5: Commit Your Files

Create your first commit:

git commit -m "Initial commit of QuickNotes project"

You should see output indicating the files that were committed.

Step 6: Add the Remote Repository

Connect your local repository to the GitHub repository:

git remote add origin https://github.com/Abdelkad3r/QuickNotes.git

This command doesn't produce any output if successful.

Step 7: Set Your Branch Name to "main"

Rename your branch to "main":

git branch -M main

This command doesn't produce any output if successful.

Step 8: Push Your Code to GitHub

Finally, push your code to GitHub:

git push -u origin main

You'll be prompted for your GitHub username and password/token.

Important: If you have two-factor authentication enabled on GitHub, you'll need to use a personal access token instead of your password. You can create one at GitHub Settings > Developer settings > Personal access tokens.

Success!

If all commands run successfully, your QuickNotes project is now available at:

https://github.com/Abdelkad3r/QuickNotes

Troubleshooting

"fatal: remote origin already exists"

If you get this error, run:

git remote remove origin

Then try adding the remote again (Step 6).

"error: failed to push some refs"

If you get this error, run:

git pull origin main --allow-unrelated-histories

Then try pushing again (Step 8).

"Authentication failed"

Make sure you're using the correct username. If you have two-factor authentication enabled, use a personal access token instead of your password.
