Pushing QuickNotes to GitHub

Note: This guide will walk you through the process of creating a Git repository for your QuickNotes project and pushing it to GitHub.

Why Use GitHub?

  • Version control - Track changes to your code
  • Backup - Keep your code safe in the cloud
  • Collaboration - Work with others on your project
  • Deployment - Deploy directly from GitHub to services like Vercel
  • Showcase - Share your work with others

Step-by-Step Guide

1 Initialize a Git Repository

Open a terminal and navigate to your QuickNotes directory:

cd MiniJobb/QuickNotes
git init

This creates a new Git repository in your project folder.

The terminal after initializing a Git repository

2 Add Your Files

Add all your project files to the repository:

git add .

This stages all files for commit.

3 Commit Your Files

Commit the files with a descriptive message:

git commit -m "Initial commit of QuickNotes project"

This creates a snapshot of your code at this point in time.

4 Create a New Repository on GitHub

  1. Go to github.com/new
  2. Name your repository (e.g., "quicknotes")
  3. Add an optional description
  4. Choose whether to make it public or private
  5. Important: Do NOT initialize the repository with a README, .gitignore, or license
  6. Click "Create repository"
Creating a new repository on GitHub

5 Connect Your Local Repository to GitHub

After creating the repository, GitHub will show you commands to push an existing repository. Run these commands:

git remote add origin https://github.com/YOUR_USERNAME/quicknotes.git
git push -u origin main

Replace YOUR_USERNAME with your actual GitHub username.

GitHub showing the commands to push your repository

6 Verify Your Repository

Go to https://github.com/YOUR_USERNAME/quicknotes to see your code on GitHub.

Your repository on GitHub after pushing your code

Using the Provided Script

For convenience, a script has been provided to help you set up Git:

./git-setup.sh

This script will initialize the repository, add all files, and commit them. You'll still need to create the GitHub repository and add the remote manually.

Important: If you're using GitHub, you might need to use a personal access token instead of your password for authentication. See GitHub's documentation for details.

Next Steps

Now that your QuickNotes project is on GitHub, you can:

  • Deploy to Vercel: Follow the Vercel deployment guide to deploy your app
  • Share with others: Send them the GitHub repository URL
  • Make changes: As you update your project, commit and push the changes to keep your repository up to date
